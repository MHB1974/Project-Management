# 🚀 3단계로 끝내는 자동 배포

복잡한 명령어 없이, **파일 올리기 → 클릭 한 번**이면 끝납니다.
GitHub가 알아서 빌드하고 배포합니다.

---

## ✅ 사전 준비 (딱 한 가지)

`src/config/firebase.js` 파일을 열어서 Firebase 정보만 넣으세요.

Firebase Console (https://console.firebase.google.com) →
프로젝트 설정(⚙️) → "내 앱" → 설정 코드 복사 → 붙여넣기 → 저장

(이미 되어 있다면 건너뛰세요)

---

## 1️⃣ 파일 올리기 (GitHub Desktop)

1. GitHub Desktop에서 Project-Management 폴더 열기
2. 이 폴더의 **모든 파일**을 그 안에 복사
   (`.github` 폴더 포함 — 숨김 폴더이니 빠지지 않게 주의!)
3. 왼쪽 아래 Summary 칸에 아무 글자나 입력 (예: "배포")
4. **Commit to main** 클릭
5. 위쪽 **Push origin** 클릭

---

## 2️⃣ GitHub 설정 한 번만 (최초 1회)

브라우저에서 본인 저장소 접속:
https://github.com/mhb1974/Project-Management

1. 위쪽 **Settings** 클릭
2. 왼쪽 메뉴 **Pages** 클릭
3. **Build and deployment → Source** 에서
   **"GitHub Actions"** 선택  ← 이게 핵심!
4. 끝. 저장 버튼도 필요 없습니다.

---

## 3️⃣ 기다리기 (2~3분)

- Push하면 GitHub가 **자동으로 빌드 + 배포**를 시작합니다.
- 진행 상황은 저장소의 **Actions** 탭에서 볼 수 있어요.
- 초록색 체크(✅)가 뜨면 완료!

배포 주소:
**https://mhb1974.github.io/Project-Management/**

---

## 🔄 앞으로 수정할 때는?

코드를 고친 뒤 → GitHub Desktop에서 **Commit → Push** 만 하면
나머지는 전부 자동입니다. (2번 설정은 다시 안 해도 됨)

빌드도, 배포도 직접 할 필요 없습니다. Push만 하세요.

---

## 🆘 막히면?

| 증상 | 해결 |
|------|------|
| Actions 탭에 빨간 X | 클릭해서 에러 확인 후 알려주세요 |
| 페이지가 비어있음 | 2~3분 더 기다린 후 새로고침 (Ctrl+Shift+R) |
| 로그인 안 됨 | firebase.js 설정 확인 |
| `.github` 폴더가 안 보임 | 숨김 파일 보기 켜기 / 압축 풀 때 누락 주의 |
